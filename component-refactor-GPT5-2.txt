Create a new component:  

WORKFLOW
- Scaffold using: npm run new <ComponentName>
- Output must be under: src/components/<ComponentName>/
- Do NOT modify the generator; if scaffolding fails â†’ FAIL
- After scaffolding, edit ONLY files in: src/components/<ComponentName>/
- Do NOT read, import, or reference any other files or components

IMPLEMENTATION
- Thin wrapper around the matching Ant Design component
- Do NOT re-implement any Ant Design logic

PROPS (HARD RULES)
- Inherit Ant Design props BUT expose ONLY:
  - Boolean props
  - Enum / union props with finite, selectable values
- NEVER expose:
  - Free-form text or content props (e.g. placeholder, labels, children)
  - Callback / event props
  - Render props
  - Object, style, or passthrough props
  - Deprecated props
  - Props without documented selectable options
- If a wrapper prop conflicts with an Ant Design prop:
  - Omit the Ant Design prop from inherited types
  - Wrapper prop is the single source of truth
  - Do NOT forward the omitted Ant Design prop
- Always include:
  - data-testid?: string

STYLING
- Minimal, scoped CSS only
- If popup styling is required:
  - Use popupClassName internally
  - Scope all popup styles to it
- Include a fixed input-width class (e.g. 240px) when relevant

STORIES / DOCS
- Use the default Plop Storybook docs template ONLY
- Do NOT add custom docs sections
- If controls are present, include ONLY allowed props

OUTPUT
- Output ONLY files under: src/components/<ComponentName>/
- No explanations, comments, or extra text
